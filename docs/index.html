The Taxonomy

Five control abstractions, from lightweight role instructions to governed agent runtimes. Each adds structure — and cost. The benchmark measures when the trade-off pays off.

L1

Persona

persona

Role and interaction style. Easy to write, good UX — but hidden assumptions, weak reproducibility, role drift.

Best for: Low-risk tasks where voice matters

L2

Lens

lens

Constrained viewpoint. Sharper than persona, less theatrical — but still no defined procedure.

Best for: Tasks needing a stable perspective

L3

Skill

skill

Procedure + output schema. Interpretable, testable, reusable. The default unit of reliable work.

Best for: Repeatable, reviewable tasks with known output shape

L4

Agent

agent

Tools + state + multi-step reasoning. Useful when tasks require search, branching, or external action.

Best for: Tasks requiring tools, state, or multi-step search

L5

Harness

harness

Guards + retry + audit trail. The evaluation and control layer. Highest implementation complexity.

Best for: Compliance, safety, production actions, audit requirements

How It Works

1

Load

Define task suites and taxonomy specs in YAML. Five treatment types, seven task classes.

2

Run

Execute every task through every treatment. N repetitions, P paraphrases. All outputs traced to SQLite.

3

Compare

Generate HTML/Markdown reports with per-treatment scores, promotion signals, and stability analysis.

Quickstart

Install, initialize, and run your first benchmark in four commands.

terminal 

$ uv sync --extra dev  # Install dependencies

$ uv run promptstackbench init  # Initialize workspace

$ uv run promptstackbench run --suite architecture_review --treatments persona,lens,skill --mock  # Run benchmark

$ uv run promptstackbench report --run-id <id> --format html  # Generate report
