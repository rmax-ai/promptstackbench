PromptStackBench

Evaluate when a persona should become a skill, when a skill should become an agent, and when an agent needs a harness.

Compare answer quality, groundedness, stability, cost, latency, and promotion value across the taxonomy.

Today the runnable benchmark focuses on persona, lens, and skill. Richer agent and harness execution models are planned rather than fully implemented.

Quickstart
uv sync --extra dev
mkdir -p workspace/readme-example
uv run promptstackbench init --path workspace/readme-example
export OPENAPI_API_KEY=<api-key>
uv run promptstackbench run \
  --suite architecture_review \
  --treatments persona,lens,skill \
  --repetitions 1 \
  --paraphrases 1 \
  --mock \
  --data-dir workspace/readme-example/datasets \
  --specs-dir workspace/readme-example/specs \
  --traces-dir workspace/readme-example/traces
uv run promptstackbench report \
  --latest \
  --format html \
  --traces-dir workspace/readme-example/traces

How It Works

Load

Define task suites and taxonomy specs in YAML, then initialize a fresh workspace directory for each example run.

Run

Execute each task through each treatment with configurable repetitions and paraphrases. Outputs are stored in SQLite.

Compare

Generate HTML or Markdown reports with scores, robustness summaries, and promotion signals.

Metrics

Answer Quality

Correctness, completeness, clarity, and relevance.

Grounding and Structure

Hallucination score, schema validity, and instruction adherence.

Robustness

Paraphrase stability and run variance.

Operational

Token usage, latency, cost, and promotion-value tradeoffs.

Workspace Convention

This repository includes an ignored workspace/ directory for disposable example runs. The documented commands create workspace/readme-example/ so they can be copied and pasted without polluting tracked project files. For real model runs, the CLI reads OPENAPI_API_KEY by default and falls back to OPENAI_API_KEY, config.yaml, or --api-key.

Current Status

Implemented Today

Persona, lens, and skill treatments run end to end through the current prompt-based benchmark pipeline.

Planned

Agent and harness specs are modeled and loadable, but distinct runtimes with tools, state, guards, retries, and audit behavior are still planned work.

Taxonomy Layers

  • Persona: role, tone, and style.
  • Lens: constrained viewpoint.
  • Skill: procedure plus output schema.
  • Agent: planned richer runtime with tools, state, and multi-step execution.
  • Harness: planned control layer with guards, retry, and audit behavior.
